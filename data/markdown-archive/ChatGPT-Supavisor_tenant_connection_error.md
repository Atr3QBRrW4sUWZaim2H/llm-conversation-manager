```yaml
    container_name: supabase-analytics
    ...
```

```

```

**Common mistake:** If your config says something like `0.0.0.0:4000`, `localhost:4000`, or `127.0.0.1:4000`, that only works **inside** the analytics container itself (or on the host machine if you mapped the port). But from the separate `vector` container, that address wonât work.

